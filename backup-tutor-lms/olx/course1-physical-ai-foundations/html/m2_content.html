Module 2 — Robot Fundamentals

Before you can train a robot, you need to understand what a robot perceives and what it can do. This module covers the minimal set of concepts every Physical AI practitioner needs — without the heavy mathematics of a traditional robotics course.

The 5 Subsystems of Any Robot

Every robot — from a simple arm to a humanoid — has these five subsystems:

  1. Perception — Cameras, depth sensors, LiDAR, microphones. What the robot sees and hears.
  2. State Estimation — Joint encoders, IMU, odometry. Where the robot's body parts are right now.
  3. Planning — Deciding what to do next. In Physical AI, this is replaced by a learned policy.
  4. Control — Translating decisions into motor commands. PID controllers, impedance control.
  5. Actuation — Motors, servos, pneumatics. The physical movement.
Sensors → State → Policy (learned) → Controller → Actuators

In Physical AI, the Policy replaces the classical Planning module. Instead of hand-coded logic, a neural network maps observations directly to actions.

State Space — What the Robot Knows

The state is everything the robot can observe at a given moment. For a robot arm, this typically includes:

  • Joint angles (e.g., 6 values for a 6-DOF arm, in radians)
  • Joint velocities
  • End-effector position and orientation (x, y, z, roll, pitch, yaw)
  • Camera images (RGB, depth)
  • Gripper state (open/closed, force)

For a mobile robot, state also includes position on a map, heading, and velocity. The state is the input to your policy model.

Action Space — What the Robot Can Do

The action is what the robot outputs at each timestep. Three common types:

Action Type What It Controls Use Case
Position controlTarget joint angleSlow, precise manipulation
Velocity controlJoint speedSmooth continuous motion
Torque controlMotor forceForce-sensitive tasks (assembly, surgery)
End-effector deltaChange in gripper positionMost VLA models use this

Most modern VLA models output end-effector deltas — small changes in the gripper's position and orientation at each timestep. This is simpler to learn than raw joint commands.

Control Frequency — Why It Matters

Control frequency is how many times per second the robot reads its state and sends a new action. Typical values:

  • 10 Hz — Slow, suitable for high-level planning
  • 50 Hz — Standard for manipulation tasks
  • 500–1000 Hz — Required for force control and fast dynamics

Your dataset must be collected at the same frequency you'll run inference. A policy trained on 10 Hz data will behave incorrectly if deployed at 50 Hz.

Degrees of Freedom (DOF)

DOF = the number of independent ways a robot can move. A standard robot arm has 6 DOF — enough to reach any position and orientation in 3D space. A humanoid has 30–50+ DOF. More DOF = more expressive, but harder to control and harder to collect data for.

The Minimal Robot for Physical AI

You don't need a humanoid to start. The minimal setup for learning Physical AI is:

  • 1 RGB camera (wrist-mounted or overhead)
  • 6-DOF robot arm with joint encoders
  • Position control at 50 Hz
  • Simple gripper (open/close)

This is exactly what the SO-100/SO-101 arm (used in LeRobot tutorials) provides. It costs ~$100 to build and is sufficient to train real manipulation policies.
