Module 7 — VLM & VLA Models

Vision-Language Models and Vision-Language-Action Models are the frontier of Physical AI. They enable robots to understand natural language instructions and act on them — without task-specific programming. This module explains how they work and how to use them.

Why Language?

Humans give instructions in language: "pick up the red cup," "move to the left shelf," "hand me the screwdriver." A robot that can understand and act on language instructions is far more useful than one that can only execute pre-programmed tasks. Language also enables zero-shot generalization — the robot can attempt tasks it was never explicitly trained on, by reasoning from language.

VLM — Vision-Language Model

A VLM takes an image and text as input and produces text as output.

Input: [image of table with objects] + "What objects are on the table?"
Output: "A red cube, a blue cylinder, and a yellow sphere."

In robotics, VLMs are used for:

  • Object detection — "Where is the red cube?"
  • Scene understanding — "Is the gripper above the object?"
  • Task grounding — "Which object should I pick up to complete this task?"
  • Failure detection — "Did the grasp succeed?"

Examples: GPT-4V, Gemini Vision, LLaVA, InternVL

VLA — Vision-Language-Action Model

A VLA extends a VLM by adding an action head — a component that outputs robot actions instead of text.

Input: [camera image] + "pick up the red cube"
Output: [joint_delta: 0.02, 0.01, -0.03, 0.0, 0.01, 0.0, gripper: 0.0]

The VLA processes the image and language together, then outputs the next robot action. This is end-to-end: no separate perception, planning, or control modules.

How VLAs Are Trained

  1. Start with a pretrained VLM — e.g., LLaVA, PaliGemma, Qwen-VL
  2. Add an action head — A small network that maps VLM embeddings to robot actions
  3. Fine-tune on robot data — Your teleoperation dataset with (image, language, action) triples
  4. Evaluate on held-out tasks — Test generalization to new instructions

The VLM backbone provides visual and language understanding. The action head learns to translate that understanding into robot movements.

Dataset Requirements for VLA Training

VLAs need more than standard behavior cloning datasets:

  • Language annotations — Every episode needs a task description
  • Task diversity — Multiple different tasks, not just one
  • Object diversity — Different objects, colors, shapes
  • Scale — Typically 2000+ episodes for meaningful generalization
  • Multi-view cameras — Wrist + overhead cameras improve performance

Limitations of Current VLAs

  • Data hungry — Need large, diverse datasets to generalize
  • Slow inference — Large models can't run at 50 Hz without optimization
  • Brittle to distribution shift — Still fail on unseen environments
  • Hard to debug — End-to-end models are black boxes

These are active research problems. The field is moving fast — expect significant improvements in 2025-2026.
