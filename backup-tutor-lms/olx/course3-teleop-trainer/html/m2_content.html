Module 2 — Designing a Data Collection Session

A well-designed session produces usable data. A poorly designed session wastes everyone's time. This module teaches you how to plan before you collect.

Task Decomposition

Before collecting data, break your task into clear, atomic steps. Example for "pick and place red cube":

  1. Start position: arm at home, gripper open
  2. Approach: move arm above the cube
  3. Descend: lower arm to grasp height
  4. Grasp: close gripper
  5. Lift: raise arm with cube
  6. Transport: move arm to target location
  7. Place: lower arm to placement height
  8. Release: open gripper
  9. Retract: return arm to home position

Each step should be consistent across all episodes. If step 3 looks different in every episode, the AI can't learn it.

Episode Length

Keep episodes short and focused. A 5-second pick-and-place is better than a 30-second wandering session. Shorter episodes = cleaner data = faster training.

  • Simple manipulation: 3-10 seconds per episode
  • Multi-step tasks: 10-30 seconds per episode
  • Navigation tasks: 30-120 seconds per episode

Reset Procedures

Every episode must start from the same (or similar) starting state. Define your reset procedure before collecting:

  • Where does the robot arm start? (home position)
  • Where is the object placed? (fixed position, or randomized within a range?)
  • What is the camera angle?
  • What is the lighting?

Write this down. Follow it every time. Consistency in resets = consistency in data.

Coverage Planning

Coverage means collecting data across the full range of conditions the AI will encounter. Plan for variation:

  • Object position: collect from 5 different positions
  • Object orientation: collect from 3 different rotations
  • Approach angle: collect from 2 different approach directions

A dataset with 100 episodes from one position is less useful than 50 episodes spread across 5 positions.
