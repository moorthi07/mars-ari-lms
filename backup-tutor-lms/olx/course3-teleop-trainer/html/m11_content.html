Module 11 — The Iteration Loop

Physical AI is not a one-shot process. It's an iterative loop of collect, train, evaluate, and improve. This final module teaches you how to run that loop effectively.

The Full Loop

1. Design session (Module 2)
2. Collect data (Modules 3-5)
3. Curate data (Module 6)
4. Label data (Module 7)
5. Train policy (Module 9)
6. Evaluate in simulation
7. Identify failure modes
8. Design targeted collection to fix failures
9. Collect more data (targeting failures)
10. Retrain with augmented dataset
11. Deploy to real hardware (with safety layer)
12. Collect HIL data (human corrects real failures)
13. Fine-tune on HIL data
14. Go to step 6

Reading Evaluation Results

After training, run 20 evaluation rollouts in simulation. For each failure, ask:

  • Where did it fail? — Which phase of the task? (approach, grasp, place?)
  • Why did it fail? — Wrong position? Wrong timing? Didn't see the object?
  • What data would fix it? — More episodes from that starting position? More variation in object placement?

Common Failure Patterns and Fixes

Failure PatternRoot CauseFix
Fails from new starting positionsNot enough position coverageCollect from 5+ different starting positions
Grasps but drops objectInconsistent grasp in training dataCollect 50 episodes focused on grasp quality
Works in sim, fails on real hardwareSim-to-real gapCollect real data, add domain randomization to sim
Oscillates near targetNoisy training data near completionCurate episodes more carefully, remove jitter
Ignores language instructionInsufficient language diversityCollect data with more varied task descriptions

When You're Done

A policy is "done" when it meets your performance target consistently. Typical targets:

  • Simple pick-and-place: 90%+ success rate in sim, 80%+ on real hardware
  • Multi-step manipulation: 80%+ in sim, 70%+ on real hardware
  • Language-conditioned: 70%+ on seen tasks, 50%+ on unseen tasks
