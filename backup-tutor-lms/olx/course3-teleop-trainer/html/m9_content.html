Module 9 — Configuring VLA & VLM Models

As a trainer, you don't build models from scratch — you configure and feed them. This module explains what you control and what you don't, and how to set up a model training run.

What a Trainer Controls

You are not a model researcher. Your job is to:

  • Prepare the dataset (Modules 2-7)
  • Select the right model architecture for your task
  • Configure training hyperparameters
  • Monitor training progress
  • Evaluate the trained policy
  • Decide when to collect more data

Key Training Hyperparameters

These are the parameters you'll typically configure:

  • Batch size — How many episodes to process at once. Larger = faster but needs more GPU memory. Start with 32.
  • Learning rate — How fast the model updates. Too high = unstable training. Too low = slow convergence. Start with 1e-4.
  • Training steps — How long to train. More steps = better performance up to a point. Monitor validation loss to decide when to stop.
  • Chunk size — For ACT: how many future actions to predict at once. Typical: 50-100 frames.

Monitoring Training

Watch these metrics during training:

  • Training loss — Should decrease steadily. If it plateaus early, you may need more data.
  • Validation loss — Should decrease with training loss. If it increases while training loss decreases, you're overfitting.
  • Success rate in sim — Run evaluation rollouts every N steps. This is the real metric.

When to Stop Training

  • Validation loss has plateaued for 10+ epochs
  • Success rate in sim has plateaued
  • Success rate in sim is >80% (good enough to deploy and collect HIL data)
