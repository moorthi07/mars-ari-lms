Module 3 — Telemetry Data

Telemetry is the stream of sensor data coming from the robot in real-time. Understanding it helps you monitor data quality, detect problems, and understand what the AI will learn from.

What Is Telemetry?

Every sensor on the robot produces a continuous stream of measurements. The platform collects and stores all of this during your sessions:

  • Joint positions — Angle of each joint in radians, updated at 50 Hz
  • Joint velocities — Speed of each joint
  • End-effector pose — Position and orientation of the gripper in 3D space
  • Battery level — Remaining power percentage
  • Speed — Linear and angular velocity (for mobile robots)
  • Signal strength — Connection quality
  • GPS/position — Location (for outdoor robots)

Reading the Analytics Charts

The Analytics panel (right side of dashboard) shows live charts of this data. As a trainer, you should monitor these during collection:

  • Speed chart — Should be smooth curves, not spikes. Spikes = jerky movement = bad data.
  • Joint angle chart — Should show smooth transitions. Flat lines = robot not moving (idle).
  • Battery chart — Should decrease gradually. Sudden drops = hardware issue.

Anomaly Detection

Watch for these warning signs in telemetry:

  • ❌ Flat joint angles for >1 second — Robot is idle. Stop and restart the episode.
  • ❌ Speed spikes >3x normal — Jerky movement. Discard this episode.
  • ❌ Battery drop >5% in 10 seconds — Hardware issue. Stop session.
  • ❌ Signal strength below 2 bars — Connection unstable. Pause collection.

TimescaleDB — How Data Is Stored

The platform stores all telemetry in a time-series database (TimescaleDB). This means you can query historical data, compare sessions, and analyze trends over time. The data is organized by robot ID and timestamp, making it easy to retrieve any episode for review or training.
