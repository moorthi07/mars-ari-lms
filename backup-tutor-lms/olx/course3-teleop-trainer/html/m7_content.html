Module 7 — Data Labeling

Labels are the supervision signal that tells the AI what each part of your data means. Good labels enable better generalization. This module covers what to label and how.

What Labels Does a VLA Need?

For a VLA model, the primary labels are already in your dataset — the actions you performed. But additional labels significantly improve training:

  • Task description — Natural language description of the full task (from Module 5)
  • Phase labels — Which phase of the task is this frame? (approach, grasp, lift, place)
  • Object labels — What objects are in the scene? Where are they?
  • Success/failure — Did this episode succeed?
  • Key frames — Which frames are most important? (moment of grasp, moment of release)

Phase Labeling

Phase labels divide an episode into meaningful segments. This helps the model understand task structure — not just "what action to take" but "what stage of the task am I in?"

Episode phases for "pick and place":
  t=0.0 to t=1.2  → phase: "approach"
  t=1.2 to t=1.8  → phase: "grasp"
  t=1.8 to t=2.5  → phase: "lift"
  t=2.5 to t=3.8  → phase: "transport"
  t=3.8 to t=4.5  → phase: "place"
  t=4.5 to t=5.0  → phase: "retract"

Object Annotation

For vision-based policies, knowing where objects are in the image helps the model learn faster. Basic object annotation includes:

  • Bounding boxes around target objects
  • Object class labels ("red cube", "blue bin")
  • Object state ("grasped", "released", "in-bin")

Tools: Label Studio (open source), Roboflow (free tier), or the platform's built-in labeling interface.

Key Frame Selection

Not all frames are equally important. Key frames are the moments that define the task:

  • The frame just before grasping (gripper approaching object)
  • The frame of successful grasp (gripper closed on object)
  • The frame of placement (object at target location)

Some training methods (like ACT) use key frames to define action chunks — sequences of actions that lead to the next key frame.

Labeling Efficiency Tips

  • Label in batches — do all episodes of one task type together
  • Use keyboard shortcuts in labeling tools
  • Label phase boundaries first (fast), then object annotations (slower)
  • Have a second person review 10% of labels for consistency
