Module 8 — Simulation for Data Generation

Real robot data is expensive and slow to collect. Simulation lets you generate thousands of episodes automatically. This module covers when and how to use simulation to augment your real data.

When to Use Simulation

Use Simulation When...Use Real Data When...
You need 1000+ episodesYou need <200 episodes
Task is well-defined and repeatableTask requires real-world nuance
You want to test policy variations quicklyYou're doing final validation
Hardware is unavailable or expensiveSim-to-real gap is critical
You want domain randomizationYou need exact real-world physics

MuJoCo — Free, Browser-Based

MuJoCo runs in your browser (no GPU needed). Best for:

  • Quick prototyping and testing
  • Manipulation tasks with simple physics
  • Learning and experimentation

Access it directly in the MarsGeo platform — no setup required.

Genesis & Gazebo — Cloud CPU (Starter+)

More capable simulators that run on cloud CPU instances. Best for:

  • More complex physics (fluid, deformable objects)
  • Mobile robot navigation
  • Outdoor environments

Requires a Starter subscription. Billed per hour of compute time.

Isaac Sim — Cloud GPU (Pro+)

NVIDIA's photorealistic simulator. Best for:

  • Sim-to-real transfer (photorealistic rendering reduces the gap)
  • Large-scale data generation (run 100 parallel environments)
  • Humanoid and complex robot simulation

Requires a Pro subscription. Significantly more expensive but produces the best sim-to-real results.

Domain Randomization in Practice

When generating sim data, always enable domain randomization:

  • Randomize object position: ±5cm from nominal
  • Randomize object color/texture: 5-10 variations
  • Randomize lighting: 3-5 lighting configurations
  • Randomize camera position: ±2cm, ±5°

This makes your sim data more diverse and reduces the sim-to-real gap when you deploy to real hardware.
