Module 10 — Managing Data Collection Jobs

As a trainer, you're often the client on TeleOp Hub — posting jobs for controllers to complete, reviewing their work, and managing the data pipeline.

Writing a Good Job Description

A clear job description produces better data. Include:

  • Task description — Exactly what the robot should do, step by step
  • Robot type — Which robot, which controls to use
  • Episode requirements — How many episodes, how long each should be
  • Quality criteria — What makes a good episode (smooth movement, specific approach angle, etc.)
  • Reset procedure — Exactly how to reset between episodes
  • What to avoid — Common mistakes to prevent

Example Job Description

Task: Pick up the red cube from the table and place it in the blue bin.

Robot: 6-DOF arm with wrist camera. Use arm joystick controls.

Episodes needed: 100 successful episodes

Episode length: 5-10 seconds

Reset: Return arm to home position. Place red cube at marked position (center of table, 30cm from arm base).

Quality criteria: Smooth approach from above. Gripper fully closed before lifting. No pauses >0.5s.

Avoid: Approaching from the side. Dropping the cube. Idle time.

Reviewing Submitted Work

When controllers submit sessions, review them before accepting:

  1. Watch 10% of episodes in the media gallery
  2. Check telemetry charts for idle time and jitter
  3. Verify task completion rate (>90% target)
  4. Check episode length consistency
  5. Accept or request revisions

Quality Gates

Set minimum quality thresholds before accepting a batch:

  • Task success rate: >90%
  • Idle ratio: <5% of frames
  • Episode length: within ±50% of target
  • Video quality: no blurry or dark episodes
